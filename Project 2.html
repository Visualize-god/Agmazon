Product Overview

SUIVI is a digital logistics platform designed to make parcel deliveries across Kenya easier and safer. It connects senders and riders through a web-based system and mobile app, enabling real-time tracking, proof of delivery, and seamless communication. With built-in payment gateways and real-time map trackers, SUIVI provides a fast, secure, and transparent delivery experience for individuals, small businesses, or local enterprises.

Problem Statement

In Kenya, delivery remains manual and unreliable, with delayed parcel delivery, poor rider coordination, and offline payments that increase the risk of fraud.

Goals & Solution

Business Goals:

  • Digitize and optimize parcel delivery operations across Kenya.
  • Build customer trust through real-time visibility and security.
  • Reduce average delivery time and improve customer satisfaction.
  • Expand across counties with scalable logistics infrastructure.

Product Solution:

  • Senders: Easily request pickups, upload parcel details, track status, and pay via M-Pesa or card.
  • Riders: Manage deliveries, get real-time route navigation, and update parcel statuses.
  • Admins: Monitor operations, assign deliveries, handle FAQs, and manage user roles.

User Persona

1. Wanjiku – Online Boutique Owner

Needs: Quick, affordable, and visible delivery

Frustrations: Unclear delivery timelines and poor customer support

2. Otieno – Rider (Boda Boda Courier)

Needs: Clear delivery assignments and routing tools

Frustrations: Navigating multiple pickup requests without coordination

3. Admin (Operations Lead)

Needs: Real-time metrics, rider status, and full system control

User Flow & Customer Journey Map

Sender Journey:

  • Request Pickup → Enter Parcel Info → Upload Images → Pay via M-Pesa → Track in Real-Time → Receive Notifications → Leave Feedback

Rider Journey:

  • Accept Assignment → Use Map/Audio Navigation → Update Status at Each Stage → Upload Delivery Proof → Mark Task Complete

Admin Journey:

  • Dashboard Overview → Assign Deliveries → Update System Configurations → Manage Users & FAQs → Generate Reports

Success Metrics

  • Number of successful deliveries per region
  • Average delivery completion time
  • % of transactions via M-Pesa
  • Weekly sender retention rate
  • Mobile app installs (post-launch)
  • Tracking feature usage rate
  • Customer support request resolution time

My Role / Tasks

  • Drafted the comprehensive PRD for development teams.
  • Defined all user roles and permissions (sender, rider, admin).
  • Collaborated with engineers to implement features.

Key Results

  • A working MVP on Flask that supported real-time parcel status updates.
  • Tested Email/Push notification flows for key delivery moments (e.g., pickup, in-transit, arriving).

Key Learnings

  • Transparency builds trust – users need consistent updates, not just tracking links.
  • Successful delivery requires structure.
  • A unified UI/UX system improves adoption across both desktop and future mobile rollouts.
